Concepts

Instruction: Intro to variables

In Physics properties are algebraically denoted by some symbol and the symbol can represent some generic value that can change based on the situation.  For example, velocity can be expressed algebraically as  v=d/t where v, d, and t are symbols representing the physical properties velocity, distance, and time. 

In programming, variables work in a similar way.  Variables are storage locations that hold information and have three properties: name, type, and value.

Variable name

The name corresponds to the symbol in physics.  The name is used to reference the storage location that holds the information.  For our initial example, we will use the symbols (v,d,t) as the variables' names.  Later in this lesson I will talk more about naming conventions.

Variable value 

The value is the information that is stored in the location.  So, when v = 10, that mean there is a storage location in memory called v and in that location there is the number 10

Variable type

Distance, velocity, and time are have numeric values.  This is one type of variables that can be stored in memory.   However, there are multiple types of values that can be stored including  Boolean (true/false), strings, categorical.  The type of variable you have because that determines the operation that you can perform on it.  For instance, multiplication makes sense on a numeric variable but not on a categorical variable.  We will talk more about non-numeric variable types in future lessons.

Instruction: Re-assigning values to variables

Variables get their namesake from the fact that their values can change. 

In the code below...

1) d is assigned the value 100 in line 7

2) d to used to calculate v in line 9

3) d is assigned a new value, 400, in line 10

Run the code to see what happens

# the next two lines should, for now, be at the top of all your scripts 
rm(list=ls())                         # clean out the environment
options(show.error.locations = TRUE)  # give line numbers on errors

# create three variables: d, t, and v
# give d and t values and use them to calculate v
d = 100   # assign d to be 100
t = 20
v = d/t   # use d to calculate v
d = 400   # re-assigns d to 400

Notice how the value of v remains 5 even though d changes. 

This is because line 9 assigns the value of d/t to v.  It does not assign the variables.

v will remain 5 until it is assigned another value.

So, if you add the line  v=d/t at the end of the script, the value of v will reflect the change to d

Instruction: Variable naming

The problem with variable names like v, d, and t is that they are not very descriptive.  It is good programming practice to give names that are descriptive so that people reading your code can more easily understand what is going on.  So, the first step would be to spell the names out, for example: velocity, distance, and time.


However, it is most likely that a program that is solving for velocity will be calculating multiple velocities.  Perhaps is it calculating the speed of both a runner and a car.  In this case, the variable names should reflect that. 

Naming Conventions

The are two common programming conventions for variables names:

1) Capitalize the first letter of every word except the first 

     examples: runnerVelocity, runnerDistance, runnerTime

2) Put an underscore (_) between each word

     examples:  runner_velocity, runner_distance, runner_time

<TRAP: Case Counts>

For this class I will use the first convention.  For your project, please choose one of these conventions.

Naming Rules

There are also a few rules for naming a variable:

1) It must start with a letter

2) It can only contains letters, numbers, the underscore (_), or dash (-)

3) There can be no spaces in the name

4) There are system reserved words you cannot use (e.g., if, else, while, TRUE, FALSE, function, next...)

Trap: Assignment vs. Equality

The equal sign ( = ) in programming plays a different role in programming than in Algebra.  

In algebra the equal sign is an equality operator saying that the two sides are equivalent to each-other.

So, in algebra, v = d/t says that v is equivalent to d divided by t

In programming, the equals sign is an assignment operator and it says that the variable on the left side will be assigned the value calculated on the right side.

So, in programming, v = d/t says that v will be assigned the calculation of  d divided by t

These two operation can result in the exact same output but often they will not.

Treating the equal sign as an equality operation

A very common error in programming is to treat the assignment operator ( = ) as an equality operator.

The following statements make sense in algebra as equality statements but will cause errors in R. 

d/2 = 100   # d/2 is not a valid variable -- you cannot "assign" a value to it (d = 100*2 is valid)
100 = d     # '100' cannot be assigned the value of 'd' (d = 100 is valid)
d/t = v     # d/t is not a valid variable (v = d/t is valid)

The error message is displayed in the console (bottom-right corner).  The error message given for the above lines is not intuitive. 

Note that the execution of the script stops when an error occurs. 

In the following example, line 9 contains an error and the lines below it never get executed

Trap: Constants in programming

The name "variable" suggests that the value can be changed and this is usually true. However, mathematical constants like π do not change, but they are still referred to as variables in programming.  In other words, there can be a storage location referred to as pi that contains the numeric value 3.1415. These are often, and confusingly, referred to as "constant variables".

Trap: Assigning nonexistent variables

The first time you assign a value to a variable a storage container is created and a value is put in it.  We call this a declaration (more about declaration in lesson 4). After a variable has been declared, it can be used in calculations and re-assigned values.  Before this point, R knows nothing about the variable-- here is an example of using a variable before it is declared.  Note: the only change in this script was moving the line v=d/t before the declaration of d and t.  

# the next two lines should, for now, be at the top of all your scripts 
rm(list=ls())                         # clean out the environment
options(show.error.locations = TRUE)  # give line numbers on errors

# create three variables: d, t, and v
# give d and t values and use them to calculate v
v = d/t
d = 100   
t = 20 

 

There is an error in the console window saying "object 'd'  not found". The reason for this error is that, on line 7, the script is asked to assign the calculation d/t to the variable v but d and t do not exist yet.

Try the following code and see how the error changes. Note: in this code v=d/t is after the declaration of d but before the declaration of t.

# the next two lines should, for now, be at the top of all your scripts 
rm(list=ls())                         # clean out the environment
options(show.error.locations = TRUE)  # give line numbers on errors

Trap: Case counts in variable names

In R, as in most scripting language, uppercase and lowercase letters are always seen as different.

So, runnersTime and runnerstime are seen as two different variable names by R

Extension: the first two lines of code

The following to lines should, for now, be put at the beginning of any script you create

rm(list=ls()) 
options(show.error.locations = TRUE)

Line 1 cleans out the environment window each time the script is executed.  This will prevent variables from creeping over...

Line 2 adds the line number to an error statement in the console

Extension: alternate assignment operator

In R there are two assignment operators, the equal sign ( = ) and the arrow ( <- ).  The two operators are completely synonymous.

In all the code above,  ( = ) can be replaced with ( <- ) with no change in functionality, as:

d <- 100
t <- 20
v <- d/t

You will see a lot of R programs written using the arrow.  In this class, I will use the equal sign because that is the standards for most other programming languages (including JavaScript and C)
