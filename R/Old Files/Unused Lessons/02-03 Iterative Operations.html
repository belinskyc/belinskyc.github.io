for() loops are designed to iterate through a vector values.  If we want to output all the high temperatures, we can use for()

rm(list=ls())                        
options(show.error.locations = TRUE)

weatherData = read.csv("data/LansingTemps.csv")
highTemps = weatherData[ ,1]
lowTemps = weatherData[ ,2]

for(count in 1:14)

Repeated Operations

One the most common statistical function to perform is the mean of a set of data.  The mean is the total value of all the data points divided by the number of data points.  For this lesson we are going to deal with data input by the user and in the next lesson, data input from a file.

The old way of doing repeated operations

In previous lessons if we wanted to ask the user for three values and find the mean of them, we would create three variables to hold the values input by the user.  The three input values would be added together and divided by three.

rm(list=ls())                         # clean out the environment
options(show.error.locations = TRUE)  # give line numbers on errors
 
fishAge1 = readline("Enter a value: ") # get input from user - save to dataPoint
fishAge1 = as.numeric(fishAge1)       # treat dataPoint as a number
fishAge2 = readline("Enter a value: ") # get input from user - save to dataPoint
fishAge2 = as.numeric(fishAge2)       # treat dataPoint as a number
fishAge3 = readline("Enter a value: ") # get input from user - save to dataPoint
fishAge3 = as.numeric(fishAge3)       # treat dataPoint as a number
 
mean = (fishAge1 + fishAge2 + fishAge3) / 3
cat("\nThe mean age of the fish is: ", mean)

This example is OK if there are a very small number of values, but becomes incredibly unwieldy with even a moderate number of values.  Imagine trying to do the same code with 20 values -- or 1000 values!

Indexing values

Whenever you have multiple variables with names that differ only by a number (dataPoint1, dataPoint2...), the numbers are most likely index value for a larger set of data.  For instance, you might have 20 fishAge values in an excel file.  When presented visually in excel you have 20 values in one column.  The individual fishAge values can be indexed by the row it is in.

In R, there is a similar way to index values using vectors.  Like a column in excel, all of the values belongs to the same variable -- in this case, fishAge. 

So we declare a variable called fishAge but we need to tell R that fishAge is going to be multiple values

fishAge = vector()

And we index fishAge using square brackets ( [ ] ). 

rm(list=ls())                         # clean out the environment
options(show.error.locations = TRUE)  # give line numbers on errors
 
fishAge = vector()
fishAge[1] = readline("Enter a value: ") # get input from user - save as first fishAge value
fishAge[2] = readline("Enter a value: ") # get input from user - save as second fishAge value
fishAge[3] = readline("Enter a value: ") # get input from user - save as third fishAge value
fishAge = as.numeric(fishAge)       # treat all fishAge values as numbers
 
mean = (fishAge[1] + fishAge[2] + fishAge[3]) / 3
cat("\nThe mean value of the fish ages: ", mean)

Just a side note: we only need to use the as.numeric() command once -- to tell the vector fishAge to treat all its values as numbers.

Using index values

The first use of vector is that we can use the index number to retrieve a specific value.  For instance, the user can retrieve a specific value for the fishAge vector

rm(list=ls())                         # clean out the environment
options(show.error.locations = TRUE)  # give line numbers on errors
 
fishAge = vector()
fishAge[1] = 5
fishAge[2] = 7
fishAge[3] = 9
 
index = readline("Which fishAge value would you like to see? ")
index = as.numeric(index)
cat("Fish Age value number", index, "is:", fishAge[index])

Iterating through multiple indexes

While the examples above show some minor advantages of indexing, they still would be unwieldy with any more than a few values.  The real advantage to using vectors with indexes is that you can execute the same command multiple times.

For example, if you want to get 3 values for fishAge, you can have one readline() command to get all 3 values.

rm(list=ls())                         # clean out the environment
options(show.error.locations = TRUE)  # give line numbers on errors
 
fishAge = vector()
for(index in 1:3)
{
  fishAge[index] = readline("Enter a value: ") # get input from user - save as first fishAge value
}
fishAge = as.numeric(fishAge)       # treat all fishAge values as numbers
 
mean = (fishAge[1] + fishAge[2] + fishAge[3]) / 3
cat("\nThe mean value of the fish ages: ", mean)

Iterative Operation: For

Whenever you see code that repeats itself as these four line did in the script above:

dataPoint = readline("Enter another value: ") # get input from user - save to dataPoint
dataPoint = as.numeric(dataPoint)       # treat dataPoint as a number
numPoints = numPoints + 1               # increase number of points by one
runningTotal = runningTotal + dataPoint # increase the running total by the data point

It is almost a guarantee that the code needs to go in a loop.  R has two types of loops: for() and while().  The rule-of-thumb is that you use for() when you know how many times the loop will run and while() when you do not know how many times the loop is going to run -- we will look at  while() in a latter lesson.

In the script above, the user was asked for four input values, so we have a loop that runs four time.  Since we know the number of time it will run, we will use a for() loop

rm(list=ls())                         # clean out the environment
options(show.error.locations = TRUE)  # give line numbers on errors
 
### Variable declarations
numPoints = 0    # running count of input data points
runningTotal = 0 # running addition of input data points
dataPoint = NULL # variable to hold the data input
mean = NULL      # mean value of the data
### End of variable declarations
for(i in 1:4)
{
   dataPoint = readline("Enter a value: ") # get input from user - save to dataPoint
   dataPoint = as.numeric(dataPoint)       # treat dataPoint as a number
   numPoints = numPoints + 1               # increase number of points by one
   runningTotal = runningTotal + dataPoint # increase the running total by the data point
}
 
mean = runningTotal / numPoints
cat("\nThe number of data points: ", numPoints)
cat("\nThe total value of the data points: ", runningTotal)
cat("\nThe mean value of the data points: ", mean)

The code above runs exactly like the previous code.  Let's look at the for(i in 1:4) statement:

The only part of for(i in 1:4) that you need to understand now is that 4 represents the number of times the loop will iterate (we will learn what the rest of the for() means in a later lesson).  Go ahead and change that number and see how the execution of the script changes.

Like the if(), the for() has curly brackets ( { } ) attached to it.  In between the curly brackets is the code attached to the for(), in other words, it is the code that executes each time the for() loop iterates.  In the above code, the code between the ( { } ) iterates four time.

For where user determines the number

We can change the number of times the for() iterates by changing the number 4 to another number.  This is the same as hardcoding the iterations but this is only useful if we know beforehand exactly how many data points the user will input. 

Usually we cannot hardcode the number of iterations beforehand because data sets have varying number of data points.  But the data set can have the number of data points and send that information to the script before sending the data.  In our case, the user can indicate the number of data points before supplying the data.

rm(list=ls())                         # clean out the environment
options(show.error.locations = TRUE)  # give line numbers on errors
 
### Variable declarations
numPoints = 0    # running count of input data points
runningTotal = 0 # running addition of input data points
dataPoint = NULL # variable to hold the data input
mean = NULL      # mean value of the data
### End of variable declarations
 
numPoints = readline("How many data points do you have? ")
numPoints = as.numeric(numPoints)
 
for(i in 1:numPoints)
{
   dataPoint = readline("Enter a value: ") # get input from user - save to dataPoint
   dataPoint = as.numeric(dataPoint)       # treat dataPoint as a number
   #  numPoints = numPoints + 1            # not needed- the user supplied this value
   runningTotal = runningTotal + dataPoint # increase the running total by the data point
}
 
mean = runningTotal / numPoints
cat("\nThe number of data points: ", numPoints)
cat("\nThe total value of the data points: ", runningTotal)
cat("\nThe mean value of the data points: ", mean)

In this script, the number of iterations (4) in for( i in 1:4 ) was replaced by the value entered by the user and save in numPoints. 

for(i in 1:numPoints)  states that the code attached to the for() will iterate numPoints number of times

Keeping count through the for() iterations

In the statement for(i in 1:5), i is a variable that is assigned a value each time the for() iterates.

The initial assigned value of i is 1

The final assigned value of i is 5

Each time the for() iterates, the assigned value of i is increased by 1

In this example, 1:5 represents the start and end values for i.  i can also be used in the code-block attached to the for() to give the number of times the for() has iterated

for(i in 1:5)
{
   cat("This is the ", i, " time through the for loop\n");
}

However, we can use any integer values as a start and end point and the for() will iterate from the start value to the end value. 

For instance, we might want to look at the 173rd through the 186th data points in a data set...

In this example, the for() iterates 14 times (from 173 to 186 inclusive).

for(i in 173:186)
{
   cat("The " , i, " value is: ??\n");
}

i is the variable name that is commonly used in programming for counted and coordinate values (where algebra commonly uses x and y, programming uses i and j).  But any variable name can be used -- in fact, your script would probably be easier to understand with a more descriptive variable name.  The following example works exactly the same as the previous example.

for(count in 173:186)
{
   cat("The " , count, " value is: ??\n");
}

We can also use the count variable in mathematical equations.  The following code finds the first 20 powers of 2.

for(count in 1:20)
{
   cat("The " , count, " power of 2 is: ", 2^count, "\n");
}

And negative numbers (and 0) can also be used:

for(count in -10:10)
{
   cat("The " , count, " power of 2 is: ", 2^count, "\n");
}
