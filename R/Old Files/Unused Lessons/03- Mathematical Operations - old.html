But we have to deal with the square.  The easiest way in script is to just multiply a second velocity

kineticEnergy = (1/2)*mass*velocity*velocity  # this does work but it is not pretty

This solution is limited and useless for powers in root form (e.g., square root, cube root). We want a more robust solution that works for all powers.  In R we use the ^ symbol to represent a power so...

kineticEnergy = (1/2)*mass*velocity^2 # much more robust solution

R does follow order-of-operations rules but sometimes it is helpful to add parenthesis to be explicit about what gets squared

kineticEnergy = (1/2)*mass*(velocity)^2 # much more robust solution

The ( ^ ) operator works for all powers including square roots, cubed roots, or mixed power (raising to the 3/2 or 5/3).

So it seems we have this in R

velocity = (2*kineticEnergy / mass)^1/2  # but there is a problem here...

Instead of raising the (2*kineticEnergy/mass) to the 1/2 power, the above code will raise (2*kineticEnergy/mass) to the first power and then divide everything by 2.  You need to be more explicit and put the 1/2 in parenthesis

velocity = (2*kineticEnergy / mass)^(1/2)  # now we are good!
