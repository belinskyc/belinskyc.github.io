We want to find the mean low temperature. We already have the number of values given by the length of the vector:

length(lowTempData), which we will assign to the variable lowTempLength

lowTempLength = length(lowTempData);

To find the sum value of all the low temperatures we need to add them all together.  One way to do this is to add the low temperature values one at a time, using for().

lowTempLength = length(lowTempData);
for( day in 1:lowTempLength )
{
  # add the indexed value in lowTempData
}

The question is: What do we add the values in lowTempData to?  We need to create a variable that will hold the total value at the end of the for().  This variable needs to be set to zero before the for() begins -- we will name the variable totalTemp.

lowTempLength = length(lowTempData);
totalTemp = 0;  # variable to hold the sum of all the low temp values
for( day in 1:lowTempLength )
{
  # add the indexed value in lowTempData to totalTemp one at a time
}

Inside the codeblock attached to the for() we keep adding the values from the vector one at a time to totalTemp:

rm(list=ls());  options(show.error.locations = TRUE);
 
weatherData = read.csv("data/LansingWeather2.csv");
lowTempData = weatherData[ ,"lowTemp"];
lowTempLength = length(lowTempData);
totalTemp = 0;  
 
for( day in 1:length(lowTempData) )
{
  totalTemp = totalTemp + lowTempData[day]; # add the indexed temp to totalTemp
  cat("Total Temp:", totalTemp, "\n");      # not needed -displays intermediate steps
}
 
meanTemp = totalTemp/length(lowTempData);
cat("The mean low temperature is:", meanTemp);

The real magic above happens in line 11:

totalTemp = totalTemp + lowTempData[day];

Remember that the right side (totalTemp + lowTempData[day]) is evaluated first and then the variable on the left side (totalTemp) is assigned the answer.

Using for() to find the mean value of all the values in a vector

So, on the first iteration of the for(), totalTemp starts at 0 and the first value from lowTempData, which is 45, is added to it. 

0+45 = 45 is assigned to totalTemp.

On the second iteration, totalTemp is equal to 45 and the second day's temperature, which is 43, is added to it. 

43+45 = 88 is assigned to totalTemp.

On the third iteration, totalTemp is equal to 88 and the third day's temperature, which is 42, is added to it. 

88+42 = 130 is assigned to totalTemp.

This repeat for 14 iterations and, at the end, totalTemp is assigned the value of all the 14 temperatures added together, which is 621.

Calculating the mean value using the total value and the number of values.

621 is divided by the number of values, lowTempLength = 14, to get meanTemp = 44.35714

Just a reminder, in the Environment Window, day has a value of 14L.  The "L" is just referring to how the number is stored in memory and, for now, can be ignored.

Calculating mean using R functions

There is a much quicker way to find the mean of a vector.  R has a built in function to calculate the mean.  In fact R has many built in functions to generate all sorts of statistics. 

The alternate code to calculate the mean of the vector lowTempData is:

mean(lowTempData);

So, the full script would be:

rm(list=ls());  options(show.error.locations = TRUE);
 
weatherData = read.csv("data/LansingWeather2.csv");
lowTempData = weatherData[ ,"lowTemp"];
meanTemp = mean(lowTempData);

Finding the mean of a vector using an R function

Obviously, this is much quicker than using a for() but I would not recommend using this method until you get comfortable using the for().  Why?  Because behind-the-scenes mean() is performing iterations using for(), so mean() is essentially a shortcut. While R has a lot of functions that can simplify many tasks, you will come across many situations where you have to develop your own code -- where understanding the basics is vital.

Finding the minimum of a vector

Another common operation we do is find the minimum of a vector.  Again, there is a simple way to do this in R:

rm(list=ls());  options(show.error.locations = TRUE);
 
weatherData = read.csv("data/LansingWeather2.csv");
lowTempData = weatherData[ ,"lowTemp"];
 
minTemp = min(lowTempData);

Finding the minimum of a vector using an R functions

But it is best to learn how this works. And we can practice mixing loops and conditional statements

Essentially, looking for a minimum values involves checking each value in a vector to see if it is less than the current minimum

lowTempLength = length(lowTempData);
for( day in 1:lowTempLength )
{
  # check each value to see if it is less than the current minimum
}

But we first need a default minimum value before we enter the for(), which I will call minTemp.  We could set minTemp to a default value that is ridiculously high, like 100.  But, a better solution minTemp to the first value in the lowTempData vector.  This will work for any temperature set -- even those with temperature above 100 degrees.

lowTempLength = length(lowTempData);
minTemp = lowTempData[1];  # the minTemp is initially the first temperature in the vector
for( day in 1:lowTempLength )
{
  # check each value to see if it is less than the current minimum
}

Next we need to use a conditional statement to check if the vector value is less than the current minimum

lowTempLength = length(lowTempData);
minTemp = lowTempData[1];  # set the minTemp to the first temperature in the vector
for( day in 1:lowTempLength )
{
  if(lowTempData[day] < minTemp)  # if the day's temp is less than the minimum
  {
    # check each value to see if it is less than the current minimum
  }
}

Then, in the codeblock attached to the if(), we assign to minTemp the temperature from the day that was less than minTemp.  This will execute every time the condition (lowTempData[day] < minTemp) is TRUE

lowTempLength = length(lowTempData);
minTemp = lowTempData[1];  # set the minTemp to the first temperature in the vector
for( day in 1:lowTempLength )
{
  if(lowTempData[day] < minTemp) 
  {
    minTemp = lowTempData[day];  # assign the new minimum temp to minTemp
  }
}

The if() checks all values in the lowTempData vector, changing the minTemp whenever it finds a new minimum.  Putting the code together...

rm(list=ls());  options(show.error.locations = TRUE);
 
weatherData = read.csv("data/LansingWeather2.csv");
lowTempData = weatherData[ ,"lowTemp"];
 
lowTempLength = length(lowTempData);
minTemp = lowTempData[1];
 
for( day in 1:lowTempLength )
{
  if(lowTempData[day] < minTemp) 
  {
    minTemp = lowTempData[day];
  }
}

Finding the minimum value in a vector using for()

Finding day with minimum value

Let's take this a step further and get information about the day that had the minimum temperature.  First we need to capture which day that was.  We do this in a similar manner to finding the minimum temperature.  We set the minTempDay initially to 1, to represent the first day, and change minTempDay's value when we find a new minimum temperature.

rm(list=ls());  options(show.error.locations = TRUE);
 
weatherData = read.csv("data/LansingWeather2.csv");
lowTempData = weatherData[ ,"lowTemp"];
 
lowTempLength = length(lowTempData);
minTemp = lowTempData[1]; 
minTempDay = 1;  # set the day with the min temp initially to 1
 
for( day in 1:lowTempLength )
{
  if(lowTempData[day] < minTemp) 
  {
    minTemp = lowTempData[day];  
    minTempDay = day;  # change the min temp day to the current day
  }
}

Now that we know the day, we can use this information to index all the other associated valued for that day

rm(list=ls());  options(show.error.locations = TRUE);
 
weatherData = read.csv("data/LansingWeather2.csv");
lowTempData = weatherData[ ,"lowTemp"];
 
lowTempLength = length(lowTempData);
minTemp = lowTempData[1]; 
minTempDay = 1;  # set the day with the min temp initially to 1
 
for( day in 1:lowTempLength )
{
  if(lowTempData[day] < minTemp) 
  {
    minTemp = lowTempData[day];  
    minTempDay = day;  # change the min temp day to the current day
  }
}
# get the date of the min temp, the precipitation on the date, and the high temp
dateOfMin = as.character(weatherData[minTempDay, "date"]);
precipOnMin = weatherData[minTempDay, "precipitation"];
highTempOnMin = weatherData[minTempDay, "highTemp"];

Using the minimum value index to find associated values for that day

After the for() is executed, minTempDay is equal to the day that holds the minimum temperature (in this case,  11).  We can use this index number to find other information from that day -- including the actual date (from the date column), the precipitation (from the precipitation column), and the high temperature (from the highTemp column).

Note: we need to use as.character() when we are dealing with string values in a vector as in

dateOfMin = as.character(weatherData[minTempDay, "date"]);

If we do not do that, R will treat the data in a strange way (that we will discuss later).

Trap: Minding your parentheses

There are multiple places in the following script where there are parentheses within parentheses (they are underlined and in yellow):

rm(list=ls())                        
options(show.error.locations = TRUE)

weatherData = read.csv("data/MissingTemps.csv")
lowTempData = weatherData[ ,3]

totalTemp = 0             
numNA = 0

for( day in 1:length(lowTempData) )
{
  if( is.na(lowTempData[day]) )
  {
    numNA = numNA + 1
  }
  else
  {
    totalTemp = totalTemp + lowTempData[day]  
    cat("Total Temp:", totalTemp, "\n")  
  }
}

meanTemp = totalTemp/( length(lowTempData) - numNA )
cat("The mean low temperature is:", meanTemp)

Parentheses are nested within each other -- just like in Algebra and it is very easy to miss or to add an extra parenthesis when writing a script.

It is pretty easy to use the Console Window to catch an extra end parentheses -- as long as we have the following line in our script:

options(show.error.locations = TRUE)

For the following script, I added an extra parentheses to line 23:

meanTemp = totalTemp/( length(lowTempData) - numNA ) )  # extra parenthesis

Line 23 has an extra end parenthesis

In this case, R gives an error and puts a pointer ( ^ ) to where is see something unexpected (an end parenthesis where there is no start parenthesis)

It is a little trickier to catch a missing end parenthesis:

meanTemp = totalTemp/( length(lowTempData) - numNA  # missing parenthesis

Line 23 has a missing end parenthesis

In this case, R goes to line 24 believing you are still inside the parentheses.  At the beginning of line 24, it see cat() and point to that as the error -- in other words, R cannot understand why you are putting cat() inside the parentheses started on line 23.
