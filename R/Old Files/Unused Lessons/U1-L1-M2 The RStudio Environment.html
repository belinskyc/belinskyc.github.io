Variables and Debugging

Purpose:

Start your first script.  Declare a variable, assign values to the variable, and use addition to change the value. Use a debugger to see the changes to the varible

Concept:

Programming, like algebra, uses variables.  These variables are given name (often something simple like "x" or "y") and values.  The values can be simple numbers or the result of an operation. 

Perhaps a calculation directly on the data like figuring out the mean value, or meta-calculations (like categorical data) that figure out the amount of something.  In either case, the answer to the calculation needs to be stored somewhere. We are used to doing math problems like this: 5+8= 13.

But programming is more like algebra where we first declare the variable we are going to sdave the in

x = 5+8   # so, x=13 at this point

Naming Conventions:

Examples:

  1. Start new script
  2. Declare a variable with a name that represents what the variable is (fishCount)

- rules for naming variable: can only use letters (upper and lowercase), numbers, and the underscore _ (no spaces- this is often used instead!); must start with a letter (upper or lowercase)

- note: case matter!  fishCount != fishcount

- naming styles:

-- all lowercase with underscores as spaces: fish_count, ocean_location

-- capital letter to start every word after the first
