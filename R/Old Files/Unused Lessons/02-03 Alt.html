Iteration (running code multiple times)

Programmers often want to execute, or iterate, similar code multiple times.  The most simplistic example is a counter.  If you wanted your script to count from 1 to 5, you could simply output the numbers one to five in order:

rm(list=ls());  options(show.error.locations = TRUE);
cat("count is ", 1, "\n");
cat("count is ", 2, "\n");
cat("count is ", 3, "\n");
cat("count is ", 4, "\n");
cat("count is ", 5, "\n");

Output numbers 1 through 5

While this works, you can see that this would get ugly quick if you wanted to count to 100.  And completely impractical for counts of 1000 or 1,000,000.  But in programming we can reduce this above code to just a few lines no matter how high the count is.

for() and vector

There are two parts to iterating the code above: for() loops and vector values

rm(list=ls());  options(show.error.locations = TRUE);
for(i in 1:5)  # run the attached codeblock from 1 to 5
{
  cat("the count is", i, "\n");
}

This line contains most of the magic

for(i in 1:5)

1) It declares i as a variable (what I will call the iteration variable) and i will take the values from the value sequence 1:5

2) The for() executes the attached codeblock 5 times -- for each value of i (1,2,3,4,5)

Each time the codeblock iterates, the i value changes (1,2,3,4,5), which changes the output of this code:

  cat("the count is", i, "\n");

Counting from 1 to 5 by iterating a codeblock five times

Vectors are at the core of most iterations.  Let's take a step back and learn how to create out own vector, and then later use these vectors with for().

Creating your own vectors

Vectors are variables that hold multiple values.  In the last lesson we created vectors by pulling columns out of weatherData  (e.g., highTempData or lowTempData).  highTempData and lowTempData each had 14 values, representing 14 days of measurements,  and we could access individual values using the subset operator (e.g. highTempData[3] is the 3rd value in the highTempData vector). 

Now we are going to manually create vectors and assign multiple values to them.

Here are two ways to do this in R:

1) We can create a variable and assign multiple values to it using the c() command (v1 and v2 in the code below)

2) We can create a sequence of numbers using the sequence operator ( : ) (v3, v4, and v5 in the code below)

rm(list=ls()); options(show.error.locations = TRUE);

v1 = c(4,-1,7,0,9);
v2 = c(34,67,1,-88,432,6,-3);
v3 = 1:5;   # sequence from 1 up to 5
v4 = -6:3;  # sequence from -6 up to 3
v5 = 24:18; # sequence from 24 down to 18

Creating vector in R using c() and the sequence operator

In the Environment Window all the values for each of the vector variable are shown. 

Looking at v4:

int[1:10] says there are 10 integer values in the vector v4

The order of the values in the vector v4 is: -6,-5,-4,-3,-2,-1,0,1,2,3

so we can access all the values in v4 using a subset operator:

v4[1] = -6   # first value in v4
v4[2] = -5   # second value in v4
v4[3] = -4   # third value in v4
...
v4[10] = 3   # last value in v4

Indexing vector values

We can index any of the vectors we just created.  The following script outputs the 3rd value of each of the 5 vector variables (v1,v2,v3,v4,v5):

rm(list=ls()); options(show.error.locations = TRUE);

v1 = c(4,-1,7,0,9);
v2 = c(34,67,1,-88,432,6,-3);
v3 = 1:5; # sequence from 1 up to 5
v4 = -6:3; # sequence from -6 up to 3
v5 = 24:18; # sequence from 24 down to 18

cat("v1[3]=", v1[3], "\n");
cat("v2[3]=", v2[3], "\n");
cat("v3[3]=", v3[3], "\n");
cat("v4[3]=", v4[3], "\n");
cat("v5[3]=", v5[3], "\n");

Note that "v5[3]" (in quotes) outputs the string "v5[3]" whereas v5[3] (no quotes) outputs the value of v5[3]

Indexing values in the vectors

For loops and iteration number

For reasons of convention, you will often see the variable in the for() named i, but any name can be given to this variable -- in the following example we will change the name of the iteration variable to myCounter.

The codeblock attached to the for() will be executed using the vector sequence 5:11. 

The first time the codeblock is executed myCounter is assigned the value 5.

The second time the codeblock is executed myCounter is assigned the value 6.

The seventh (and last) time the codeblock is executed myCounter is assigned the value 11.

rm(list=ls());  options(show.error.locations = TRUE);
for(myCounter in 5:11)
{
  cat("the count is", myCounter,"\n");
}

Iterating a sequence from 5 to 11

Notice that changing the name of the iteration variable (myCount) did not change the execution of the script. 

Using the iteration variable

Each time the for() iterates, the iteration variable, myCount, is assigned a new value (fig), and the iteration variable can be used just like any other variable.  The following example performs a mathematical operation (multiply by 10) on the iteration variable inside the for().

rm(list=ls());  options(show.error.locations = TRUE);
for(iterationVal in -4:3)
{
  cat("Value of iteration variable: ", iterationVal, "\n");
  cat("iteration variable multiplied by 10:", iterationVal*10, "\n");
}

Note the sequence goes from -4 to 3 and has 8 values (-4, -3, -2, -1, 0, 1, 2, 3)

Iterating through the sequence and multiplying each number by 10

Running the sequence backwards

We can run the sequence backwards...

rm(list=ls());  options(show.error.locations = TRUE);
for(iterationVal in 24:16)
{
  cat("Value of iteration variable: ", iterationVal, "\n");
  cat("The inverse of the iteration value is:", 1/iterationVal, "\n");
}

In this case the iteration value starts at 24 and iterates down to 16 -- meaning the codeblock run 9 times (24,23,22,21,20,19,18,17,16)

Iterating the sequence backwards - from 24 down to 16

Changing the iteration values

We can also manually declare values for the iteration variable in the for() by creating a vector using c().

The following code has the iteration variable cycle seven times for the seven values inside the vector created by c().

rm(list=ls());  options(show.error.locations = TRUE);
for(itVal in c(6, 17, -1, 4, -9, 4.2, 0.0012) )
{
  cat("Value of itVal: ", itVal, ", and ");
  cat("itVal squared = ", itVal^2, "\n");
}

Notice that we can use negative numbers, whole numbers, and decimal numbers in the same vector. 

Iterating the manually entered vector values

for() and vectors from a DataFrame

An example of how for() really shines is it's ability to iterate through values in a data vector using the iterating variable as an index. In the previous lesson, we pulled the highTemp and lowTemp data from weatherData and saved them both as vectors.  We can use for() to iterate through and output all the values in a vector using the subsetting operation ( [ ] ).  Note that "highTemp" and "lowTemp" are the column header values in weatherData.

rm(list=ls());  options(show.error.locations = TRUE);
 
weatherData = read.csv("data/LansingWeather2.csv")
highTempData = weatherData[ ,"highTemp"];  # high temperature vector
lowTempData = weatherData[ ,"lowTemp"];    # low temperature vector

for(day in 1:14)
{
  cat(day, "- ");
  cat("high:", highTempData[day]);
  cat(", low:", lowTempData[day], "\n");
}

It the above code, the for() loops through all the values in highTempData and lowTempData and day, the iterating variable, becomes the index for the vectors.  day goes through the vector sequence 1:14, representing the 14 days of weather data in highTempData and lowTempData. 

Output the values from the vectors highTempData and lowTempData

Getting the length of the vector

We often do not know the length of the vector beforehand because that requires knowing how much data we input from the CSV file.  In other words, we don't know how many days of weather data we have.  But we can have R determine the number of days by determining the length of the vector.

rm(list=ls());  options(show.error.locations = TRUE);
 
weatherData = read.csv("data/LansingWeather2.csv");
highTempData = weatherData[ ,"highTemp"]; 
dataSize = length(highTempData);  # get length of vector

for(day in 1:dataSize)
{
  cat("high temp:", highTempData[day], "\n");
}

finding the length of a vector

The code

dataSize = length(highTempData);

assigned the length of the highTempData vector (14) to the variable dataSize (shown in the Environment Window)

and we use the variable dataSize in the for() as the last value in the vector sequence

for(day in 1:dataSize)

So the above script works if we recorded 14 days of high temperatures, 24 days, or 2,400,000 days. 

Using for() to subtract vector values

Let's say we want to figure out the difference between the daily high and low temperatures. 

Like before we can do the difference calculation iteratively, once for each day.

rm(list=ls()); options(show.error.locations = TRUE);

weatherData = read.csv("data/LansingWeather2.csv");
highTempData = weatherData[ ,"highTemp"];
lowTempData = weatherData[ ,"lowTemp"];

dataSize = length(highTempData); # same as length(lowTempData)

for(day in 1:dataSize)
{
  cat(highTempData[day] - lowTempData[day], "\n");
}

Line 8 could also be

dataSize = length(lowTempData);

as a proper data frame must have the same number of values in each column.

Line 12

  cat(highTempData[day] - lowTempData[day], "\n");

subtracts the lowTempData value from the highTempData value indexed by the iteration variable day.

Output the daily difference between the high temperatures and low temperature

Creating a vector and saving values to it

The last step is to save the difference in temperature to a vector.  In the following example, changeInTemp is declared as a vector and is iteratively assigned the difference between the highTempData and lowTempData values

rm(list=ls()); options(show.error.locations = TRUE);

weatherData = read.csv("data/LansingWeather2.csv");
highTempData = weatherData[ ,"highTemp"];
lowTempData = weatherData[ ,"lowTemp"];

dataSize = length(highTempData);
changeInTemp = c();  # declares changeInTemp as a vector

for(day in 1:dataSize)
{
  changeInTemp[day] = highTempData[day] - lowTempData[day];
}

Assigning the difference of two vectors to a third vector

The line

changeInTemp = c();

declares changeInTemp as a variable with multiple values (i.e., a vector). 

Without the above line, this line:

changeInTemp[day] = highTempData[day] - lowTempData[day];

will cause an "Object not found" error

Object not found error because the variable was not declared as a vector.

An R shortcut for subtracting and saving vector

R makes it easy to subtract values from a vector and save it to another vector.  You can treat the vectors as if they were single-value variables and do this:

rm(list=ls()); options(show.error.locations = TRUE);

weatherData = read.csv("data/LansingWeather2.csv")
highTempData = weatherData[ ,"highTemp"];
lowTempData = weatherData[ ,"lowTemp"];

changeInTemp = highTempData - lowTempData;

Creating a new vector and assigning values to it

Essentially, this line of code

changeInTemp = highTempData - lowTempData;

is doing the same thing as:

dataSize = length(highTempData);
changeInTemp = c();           

for(day in 1:dataSize)       
{
   changeInTemp[day] = highTempData[day] - lowTempData[day];
}

While you would probably use the one line version in your code, it is important to understand the long version because it contains many programming components you will need to use is other places:

1) finding the length of a vector (line 1)

2) declaring a variable is a vector (line 2)

3) iteratively performing mathematics on vector values (lines 4, 6)

4) saving values to a vector (line 6)
