3) Your first programming topic: Commenting

We are not going to spend any time on the R code itself in this lesson.  But I want you to focus on an important feature of all programming languages called commenting.  Commenting is text in an RScript that does not get executed -- it is often there to explain to the reader what is happening in the code.  In other words, it allows programmers to explain their code in English.  Notice how the functionality can still be generally understood just by reading the comments.

In the R language, comments are created using the # key.  Any text on the line after the # will be treated by R as a comment and it will not be executed.

Notice how RStudio uses a different color to delineate comments.  In the pictures above, the comments are in a lighter green.


4) Choosing a color scheme

I am not a big fan of the default color scheme in RStudio.  It does not create enough differentiation between the different components of a program.  For instance, comments (red arrows) are in green and quoted items (blue arrows) are in just a slightly different green.

 

A good color scheme can really help a programmer by helping them quickly identify parts of a program and some common errors (like misplaced quotes).  To change the color scheme

1) click on "Tools" in the main menu (circled in picture below)

2) choose "Global Options"

3) When the "Global Options" window open (in picture below), click on "Appearance"

The picture above shows the "Vibrant Ink" color scheme.  I prefer "Vibrant Ink" because I find the black background to be a little less straining on the eyes and the text colors are well differentiated. Notice how the comments (in purple) are now clearly distinguished from the quotes (in green). 

You can choose from around 20 themes in the Editor Theme window and you can change it anytime without affecting anything else.
