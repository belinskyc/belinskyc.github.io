Decision Points

In the last lesson we looked at the if() statement and how it changes the flow of a script by creating decision points.  The decision point is based on a conditional statement inside the parenthesis ( ) of the if().  The code inside the curly brackets {   } attached to the if() gets executed only if the conditional statement is TRUE. 

The next step is to execute code if the conditional statement is FALSE.  This is done using the if-else structure. 

if (<some condition is TRUE>)
{
   # execute the code between these brackets
}
else # <some condition is FALSE>
{
# execute the code between these brackets
}

Now if the condition is TRUE, the code in the curly brackets attached to the if() gets executed. If the condition is FALSE, the code in the curly brackets attached to the else gets executed.

If-else statements and conditional operators

Using the same example from last lesson, we will ask the user their age and, this time, will give a response for both a TRUE condition and a FALSE condition.

rm(list=ls())
options(show.error.locations = TRUE)
yourAge = readline("What is your age? ") # but, R will treat this as a string
yourAge = as.numeric(yourAge) # you need to be explicit that this is a number
if(yourAge < 20)  # yourAge is greater than 20
{
   cat("You are only ", yourAge, "? You have your whole life ahead of you!!")
}
else # yourAge is not greater than 20 (it is less than or equal to 20)
{
    cat("You are quite the grown up!")
}

If you run this script then the message attached to the if() will be given if the age is less than but not equal to 20, and nothing is outputted if the age is greater than or equal to 20.   

If-Else statements using strings

Conditional operators can also be used to compare two string values.  Let's change the above script to one that asks the user for a input that is a string.

rm(list=ls())
options(show.error.locations = TRUE)
favCheese = readline("What is your favorite type of cheese? ") 
if(favCheese == "Muenster")   # favCheese equal to "Muenster"
{
  cat("You are a culinary genius!!")
}
else   # favCheese not equal to "Muenster"
{
  cat(favCheese, "?!? You have much to learn!")
}

The first message will only be outputted to the console if the user entered "Muenster" as the answer -- entering "muenster", "meunster", and "Meunster" will result in the second message being displayed.  In  the next lesson we will look at how to check for misspellings.

Note that the conditions in the above code can be reverse and the code will run exactly the same

rm(list=ls())
options(show.error.locations = TRUE)
favCheese = readline("What is your favorite type of cheese? ") 
if(favCheese != "Muenster")   # favCheese not equal to "Muenster"
{
   cat(favCheese, "?!? You have much to learn!")
}
else   # favCheese equal to "Muenster"
{
cat("You are a culinary genius!!")
}

Adding in more conditions: If-Else-If Structures

Using if-else structures, we are limited to two conditions.  However, the examples above contains situations that could easily have more than two conditions.  For instance, we might want to check more cheese types than just muenster.  Perhaps we want to output a message if the user types in "swiss" or "american". 

In the code above, the else statement is executed in all situations where favCheese is not "Muenster".  We can break down the else statement into more cases by using the if-else-if structure.

rm(list=ls())
options(show.error.locations = TRUE)
favCheese = readline("What is your favorite type of cheese? ") 
if(favCheese == "Muenster") { 
   cat("You are a culinary genius!!")
}
else if(favCheese == "Swiss") {
cat("!") zxcv
}
else if(favCheese == "American") {
cat("Is there any real milk in there?")
}
else if(favCheese == "Gouda") {
cat("So you like the smell of dirty socks?")
}
else{ # all the above options were false
   cat(favCheese, "?!? You have much to learn!")
}

TRAP: TRUE/FALSE are keywords, T and F are not <might move to a different lesson>

The terms TRUE and FALSE are reserved keywords in R (and most programming languages) -- this means that TRUE and FALSE cannot be used as variable names in R.  You will often see R scripts that use T and F as shortcuts for TRUE and FALSE.  R accepts T and F as shortcuts for TRUE and FALSE but T and F are not keywords.  This means that someone can assign a value to T and F that override the default TRUE/FALSE values. If this happens, using T or F as shortcuts for TRUE or FALSE would produce unexpected results.  It is best to stick with the reserved (and protected) keywords TRUE and FALSE.

TRAP: String/Number comparisons

If you forget to explicitly say that yourAge is a number, R will still attempt to make a comparison between yourAge and 20 in line 6 in the code below. 

rm(list=ls())
options(show.error.locations = TRUE)
yourAge = readline("What is your age? ") # but, R will treat this as a string
if(yourAge < 20)  # check the condition: is yourAge greater than 20?
{
   # if the condition is TRUE, execute the following
    cat("You have your whole life ahead of you!!")
}

The result seems to work at first if you use values between 1 and 99 but if you look a little deeper, many problems emerge.  For instance the values 1111, !!, or "hello" (with the quotes) are all considered to be less than 20.  This is because R converts the number 20 to a string in order to compare it to the string that was entered by the user (even if the string has only numbers).  The rules for determine whether one string is greater than another string is complicated and has to do with the hex-value that represent the string <an EXTENSION??>.

Needless to say, this is not the kind of functionality you want.  Most other scripting language will give you a warning or an error if you attempt to make a comparison between a number and a string.  In R, you just need to be careful about explicitly calling something a number.

EXTENSION: Skipping brackets in if-else statements

You will often see if-else statements with no brackets.  This can only be done of there is only one line of code attached to the if() or the else.

if(favAnimal == "Llama")
   cat("You are wise beyond your years")
else
   cat("You still have a lot of room for growth")

The code above is equivalent to the code below

if(favAnimal == "Llama")
{
   cat("You are wise beyond your years")
}
else
{
   cat("You still have a lot of room for growth")
}

The only advantage to the first example is that it takes up less space.  I generally recommend that you put in brackets even with only one line of code -- it is more explicit and you are often going to later on add more lines of code to the statements.
